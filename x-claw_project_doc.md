### 2.2 核心组件详解

#### 2.2.1 X-Claw Gateway(接入网关)

**职责边界**:
- 不负责业务逻辑,只做"翻译官"
- 将飞书/Teams 的专有协议转换为内部标准格式
- 处理平台特定限制(消息长度、频率、格式)

**关键设计**:
- 内部标准消息格式: platform, user_id, session_id, content, msg_type, timestamp, metadata
- 支持协议: Feishu Webhook/Event, Teams Bot Framework

#### 2.2.2 X-Claw Core(编排引擎)

**核心职责**:
1. **会话生命周期管理**: 创建、恢复、超时清理
2. **记忆检索与注入**: 根据查询自动检索相关记忆
3. **Agent Loop 控制**: 管理 MiroThinker 的思考-行动循环
4. **工具调用编排**: 决定何时调用、如何组合、结果处理

**关键算法 - Agent Loop**:
- 初始化消息历史(系统提示 + 用户查询 + 相关记忆)
- 循环调用 MiroThinker(最多 50 次迭代)
- 如果模型请求工具调用: 并行执行 → 结果注入上下文 → 继续循环
- 如果模型给出最终答案: 返回结果
- 动态调整: 迭代过多时提前总结

#### 2.2.3 Hybrid Memory System(分层记忆系统)

**四级记忆架构**:

| 层级 | 存储介质 | 容量 | 保留时间 | 检索方式 | 作用 |
|------|---------|------|---------|---------|------|
| **L1 工作记忆** | GPU 显存 | 256K tokens | 当前对话 | 直接注入上下文 | 维持当前话题连贯性 |
| **L2 会话记忆** | Redis | 10轮对话 | 24小时 | 精确匹配 | 短期上下文恢复 |
| **L3 主题记忆** | ChromaDB | 无限制 | 永久 | 语义相似度检索 | 跨会话知识关联 |
| **L4 知识图谱** | Neo4j | 百万级实体 | 永久 | 图遍历查询 | 结构化知识推理 |

**记忆流动机制**:
```
用户输入 → L1(当前上下文)
    ↓
L1 溢出/总结 → L2(Redis 短期)
    ↓
L2 过期/归档 → L3(ChromaDB 语义)
    ↓
L3 实体提取 → L4(Neo4j 图谱)
    ↓
新知识 ← 检索增强 → 未来对话
```

**创新点 - 主动记忆决策**:
- 提取关键事实(从对话中)
- 评估记忆价值(信息密度、时效性、独特性、可复用性)
- 高价值记忆存入 L3 主题记忆
- 提取实体关系更新 L4 知识图谱
- 定期记忆巩固(回顾重要记忆)

#### 2.2.4 MCP 客户端层

**MCP (Model Context Protocol)** 是 Anthropic 提出的开放标准,已成为工具连接的事实标准。

**架构优势**:
- **进程隔离**: 每个工具独立进程,崩溃不影响主系统
- **语言无关**: 工具可用任何语言实现(Python/Node/Go)
- **动态发现**: 运行时自动发现可用工具
- **类型安全**: JSON Schema 定义接口

**X-Claw 的 MCP 管理**:
- 工具生命周期管理(启动、连接、健康检查、自动恢复)
- 支持 stdio 和 sse 传输方式
- 工具结果缓存与去重
- 调用编排(串行、并行、条件)

### 5.2 重点投入领域详解

#### 领域 1: 记忆系统(持续最高优先级)

**为什么最重要?**
- 这是 X-Claw 与所有竞品的根本差异
- 使用越多,价值越高(数据飞轮)
- 技术壁垒高,难以复制

**投入计划**:

| 阶段 | 投入内容 | 预期效果 |
|------|---------|---------|
| **当前** | L1-L3 基础实现 | 支持基础记忆 |
| **Q2** | L4 知识图谱 + 主动记忆 | 自动知识沉淀 |
| **Q3** | 记忆冲突解决 + 个性化 | 越用越懂你 |
| **Q4** | 跨用户知识联邦 | 组织智慧共享 |

**技术细节 - 主动记忆评估**:
评估记忆价值的多维模型:
1. 信息密度 (实体数量 / 文本长度)
2. 时效性 (是否涉及未来计划)
3. 独特性 (与已有记忆的差异度)
4. 可复用性 (是否可能在未来被检索)

加权计算综合得分,高于阈值则存入长期记忆

#### 领域 2: 工具生态(第二优先级)

**策略**: 核心工具自研 + 通用工具集成

**工具路线图**:

| **P0** | 浏览器 (Playwright) | 网页解析 | 集成 MiroFlow |
| **P0** | 文件读取 | 文档解析 | 集成 MiroFlow |

| **P1** | 代码仓库 | 技术文档分析 | 自研 MCP |

| **P2** | 日历系统 | 日程管理 | 自研 MCP |
| **P3** | 行业专用 | 金融/法律/医疗 | 合作伙伴 |

#### 领域 3: 性能优化(持续投入)

**优化方向**:

1. **推理加速**
   - 量化技术: FP8 → INT8(精度允许时)
   - 投机采样: Draft model 加速
   - 前缀缓存: RadixAttention 复用

2. **并发提升**
   - 动态批处理: Continuous Batching
   - 多实例负载均衡
   - 请求优先级调度

3. **成本降低**
   - 模型蒸馏: 30B → 8B(简单任务)
   - 边缘缓存: 热点查询缓存
   - 智能降级: 高峰期简化响应
